# MovieHub 产品需求文档（PRD）

## 1. 产品名称

MovieHub：电影内容管理与发现平台

---

## 2. 产品概述

MovieHub 是一个基于 Flask 开发的轻量级电影内容管理与发现平台。

平台支持电影信息管理、CSV 数据集导入、搜索筛选、REST API 数据访问、Dashboard 数据看板、Watchlist 待看列表，以及基于规则的相似电影推荐功能。

本项目作为个人作品集项目进行重构和扩展，主要用于展示全栈开发能力、数据库设计能力、API 设计能力、数据管理能力，以及产品需求拆解和功能规划能力。

---

## 3. 产品背景

电影内容平台和媒体资料库通常需要管理大量结构化数据，例如电影标题、导演、类型、评分、上映年份、简介和海报链接等信息。

在小型内容管理场景中，用户或内容管理者通常需要一个简单清晰的系统来支持：

* 高效创建和维护内容数据
* 批量导入电影数据
* 快速搜索、筛选和排序电影记录
* 查看基础内容数据指标
* 记录用户感兴趣的内容
* 基于当前内容发现相似电影

MovieHub 通过一个小型但完整的内容管理与发现平台，模拟了内容产品中的核心业务流程。

---

## 4. 目标用户

### 4.1 普通电影浏览用户

希望浏览、搜索、收藏和发现感兴趣电影的用户。

### 4.2 内容管理人员

负责维护电影数据、导入数据集、管理电影资料库内容质量的用户。

### 4.3 产品经理 / 数据分析人员

希望通过 Dashboard 了解电影库内容结构、评分情况、类型分布和用户兴趣信号的用户。

---

## 5. 用户痛点

### 痛点 1：电影查找效率低

当电影数量增加后，用户很难快速找到自己感兴趣的电影。

### 痛点 2：手动录入数据效率低

内容管理人员如果逐条录入电影信息，会消耗大量时间。

### 痛点 3：缺少内容数据洞察

如果没有 Dashboard 数据看板，内容管理者难以快速了解电影库的内容结构和质量。

### 痛点 4：缺少用户兴趣记录

如果没有 Watchlist 等功能，系统难以捕捉用户对电影内容的兴趣信号。

### 痛点 5：内容发现能力较弱

用户浏览某部电影后，系统如果不能推荐相似电影，会降低继续浏览和发现内容的机会。

---

## 6. 产品目标

MovieHub 的核心目标包括：

* 提供简单可用的电影内容管理系统
* 支持电影搜索、筛选和排序，提高内容查找效率
* 支持 CSV 数据批量导入，提高数据初始化效率
* 提供 REST API，支持未来前端或外部客户端调用
* 提供 Dashboard 数据看板，展示基础内容指标
* 通过 Watchlist 记录基础用户偏好
* 通过相似电影推荐提升内容发现体验

---

## 7. 核心功能

### 7.1 电影信息管理

用户可以：

* 新增电影记录
* 查看电影列表
* 查看电影详情
* 编辑电影信息
* 删除电影记录

### 7.2 搜索、筛选与排序

用户可以：

* 根据电影标题搜索电影
* 根据电影类型筛选电影
* 按评分排序
* 按上映年份排序

### 7.3 CSV 数据集导入

内容管理人员可以：

* 从 CSV 文件导入电影数据
* 一次性导入多条电影记录
* 自动跳过重复电影标题
* 将年份和评分字段转换为合适的数据类型

### 7.4 REST API

系统提供以下 API：

* `GET /api/movies`
* `GET /api/movies/<id>`

API 以 JSON 格式返回电影数据，为未来前端扩展或外部系统调用提供基础。

### 7.5 Dashboard 数据看板

Dashboard 页面展示：

* 电影总数量
* 平均电影评分
* 最高评分电影
* 最常见电影类型
* 不同电影类型的数量分布

### 7.6 Watchlist 待看列表

用户可以：

* 将电影加入 Watchlist
* 在独立页面查看已加入 Watchlist 的电影
* 从 Watchlist 中移除电影

### 7.7 相似电影推荐

系统根据以下规则推荐相似电影：

* 与当前电影类型相同
* 排除当前正在浏览的电影
* 按评分从高到低排序
* 最多展示 3 部推荐电影

## 8. 用户故事

### 用户故事 1：电影搜索

作为电影浏览用户，我希望能够根据电影标题搜索电影，从而快速找到自己感兴趣的内容。

### 用户故事 2：类型筛选

作为电影浏览用户，我希望能够按照电影类型筛选内容，从而更方便地浏览自己喜欢的电影类别。

### 用户故事 3：内容管理

作为内容管理者，我希望能够新增、编辑和删除电影记录，从而高效维护电影资料库。

### 用户故事 4：CSV 数据导入

作为内容运营人员，我希望能够从 CSV 文件批量导入电影数据，从而避免手动逐条创建电影记录。

### 用户故事 5：Dashboard 数据分析

作为产品经理，我希望能够查看电影数量、平均评分、类型分布等指标，从而理解电影库的内容结构和质量。

### 用户故事 6：Watchlist 待看列表

作为用户，我希望能够将感兴趣的电影保存到 Watchlist，从而之后可以方便地再次查看。

### 用户故事 7：相似电影推荐

作为用户，我希望在电影详情页看到相似电影推荐，从而发现更多相关内容。

| 功能模块      | 需求描述                   |
| --------- | ---------------------- |
| 电影增删改查    | 系统应支持电影记录的新增、查看、编辑和删除。 |
| 搜索功能      | 系统应支持根据电影标题进行关键词搜索。    |
| 类型筛选      | 系统应支持根据电影类型筛选电影。       |
| 排序功能      | 系统应支持按照评分和上映年份排序。      |
| CSV 导入    | 系统应支持从 CSV 文件批量导入电影数据。 |
| REST API  | 系统应以 JSON 格式提供电影数据接口。  |
| Dashboard | 系统应展示基础电影数据指标。         |
| Watchlist | 系统应支持用户保存和移除感兴趣的电影。    |

## 10. 非功能需求

### 易用性

界面应保持简洁清晰，用户可以快速理解主要操作。

### 可维护性

项目结构应清晰，便于后续扩展和维护。

### 数据一致性

CSV 导入时应避免重复电影记录。

### 性能

搜索、筛选和 Dashboard 查询应能支持小到中等规模的数据集。

### 可扩展性

系统应能在未来扩展用户登录、评论评分、个性化推荐、网页端 CSV 上传和云端部署等功能。

## 14. 产品复盘

MovieHub 最初可以被理解为一个简单的电影数据库项目，但在重构过程中，它被扩展成了一个更偏产品化的内容管理与发现平台。

项目不仅包含 CRUD、数据库设计、REST API 等技术功能，也加入了 Dashboard 数据看板、Watchlist 用户行为记录和相似电影推荐等产品功能。

因此，该项目可以作为技术型产品经理、AI 产品经理或数据产品经理方向的作品集项目，展示从需求拆解、功能规划到技术落地的完整过程。
